Important Concepts

Lesson 1: Basics of the Web

The World Wide Web (WWW) is a collection of HTML docs (HTML = hyper text markup language), that glue everything together through the use of links (hyper links), thus creating the web. Invented in the early 90s, there are now 30+ billion pages.

Several major "pieces" of the web are inter-related and work together, including You :), a computer running a browser, the internet, and servers. The internet and servers talk to each other through http.

HTML is short for hyper text

markup language, which is made up of... text content - what you can see; markup - what it looks like; references to other docs or files with associated attributes (images/videos, etc.), and links to other pages.

HTML markup is made of tags. There is an opening tag, followed by content, and then a closing tag. Each tag is named by a tag description within brackets. An entire tag is called an element. HTML attributes are tag attributes that ascribe a value, and tags can have multiple values. An example of an attribute assigned to a tag is an anchor. Anchors are for making links, like assigning a URL as an attribute to piece of content.

There is specific nomenclature associated with defining an HTML Document. Doctype specifies the type of document at the beginning, followed by the opening doc tag, like html. This indicates the beginning of the document. After this identification, attributes like head, title, etc., are included and this is where CSS guidelines would be defined. After the head and title tags, the body of the document is coded.

Image tags don't have a closing tag and are called a void tag if they don't have a closing tag, because these kinds of tag have no content. Good web coding citizenry occurs when coders assign alternate text to display when an image doesn't load, even though doing so isn't required. It's also important to remember that all line spaces must be defined, forcing breaks through the use of tags.

Inline or block tags are used to simply end a line of text, and are void tags like line break = br. Block tags make an invisible box for the text, and these can be assigned height and width. Span is an example of an inline tag that represents text, and div is an example of a block tag that behaves like a paragraph tag and creates a box around the tagged section.

Lesson 2 - Basics of Learning to Code

How Learning to Code Works

Whenever we learn anything, we begin as novices. Learning is a continuum of moving from being novices to becoming experts. The goal of this course is for learners to gain ability in most fundamental programming concepts, and awareness of many concepts beyond this class.

As learners, we go through stages of expertise, including...

  • Ignorance - In this stage, a novice could feel like they lack any awareness or understanding of the topic, but this is where every journey to master begins.
  • Awareness - Learners can move quickly from Ignorance to Awareness.
  • Ability - It takes a little more time to reach ability, but it is at this stage that learners begin to use their newfound, possibly still rudimentary, skills.
  • Fluency - It takes the longest amount of time to reach ths level, but it is here that learners have reached expert status.

How the Web Works/How HTML is Structured

Front-end web development is visual. You get to SEE what you develop in a browser as you create it, so there's immediate gratification and early rewards. Yay! :) HTML, CSS, and Java script have different purposes when developing a web page. Each purpose is related to how you break down a visual design to create a web page:

  • Structure - HTML is like the structure of a house
  • Design - CSS is like decorating the house... the wall color, artwork, architectural details, etc.
  • Modification - Java script is like any changes you make to the house, like when a garage door opener changes the structure of the house by opening the garage door

But wait, how does the browser read this?

Code has a tree-like structure. Instructions for the browser are found in HTML's tree-like structure. While HTML and CSS are both languages, HTML controls the structure of the web page while CSS controls the style of the page (how it looks). When a programmer talkes about the DOM (Document Object Model), they are talking about the tree-like structure of the page - which is how the browser understands the page.

There are shared attributes as well as differences across HTML, CSS and DOMs. HTML has language, syntax and rules; as does CSS. HTML uses tags to describe content, which can be empty or contain another element, and tags can ascribe attributes and values in addition to describing content. The tree-like structure of HTML provides the structure and content of the page from the outermost variable to the innermost.

HTML, CSS, and Dom

Similar to HTML, CSS also has language, syntax and rules, but uses it's language to designate the style aspects of the page - i.e., what style to use and where to get it. Unlike HTML and CSS, DOM doesn't get down to the level of describing the content or style of the page, but is instead the standard convention for interacting in HTML and describes the elements in the tree.

  • HTML - Hypertext Markup Language
  • CSS - Cascading Style Sheet; style sheet language used for describing the look and formatting of a document written in a markup language
  • DOM - Document Object Model; a cross-platform and language independent convention for representing and interacting with objects in HTML (and other markup languages). The nodes of each document are organized in a tree structure, called a DOM tree.

Lesson 3: Applying CSS to Create Visual Structure in HTML

CSS stands for Cascading Style Sheets. It provides a framework to apply style to elements within HTML code and bring design to the raw structure of an HTML document.

The application of CSS to an HTML document is like the way we handle governance within the US model. Laws, or governance, is provided at many levels - national, state, municipality... you even have "rules" within your home. Each type of governance determines a different level of "rule." Similar to how nesting dolls stack within each other, the same concept of the variable structure of laws/governance from the most local (home) to the highest (federal), CSS is applied using a concept called Inheritance. In CSS, inheritance means the most local "rule" wins. For example - if we applied CSS to our laws in the US, our house rules would run the country! Another way of thinking about this is that the most specific rule in CSS applies to each element. You keep "cascading" through the rules until you find the one that applies most specifically to a given element.

How does CSS "control" HTML?

CSS is code written to tell HTML code how it should look, or represent, on a web page. It does this by specifying color, size, and other attributes of each element. Further differentiation among elements can be applies using CSS classes.

CSS is the application of a concept in software engineering called "don't repeat yourself", or DRY. This concept is aimed at reducing repitition of information of all kinds and states that "Every piece of knowledge must have a single, unabmiguous, authoritative representation within a system." When applied successfully, this principle allows single elements within a system (or body of code) to be changed without impact to other logically unrelated elements. Likewise, defining the style for elements that are logically related - like the definitions for style in a CSS file that dictate visual representation for similar classes of elements - makes sure that a single update to the specificaion in the CSS file will result in all logically related elements staying in sync and updating accordingly. The coder makes a single change, and the entire body of HTML code will reflect the change uniformly. This makes for efficient code and avoids repetition.

Using CSS with Web Pages

There are three ways to incorporate style into HTML documents using CSS. First, you can write CSS into the head tag. This option would work for small documents, with minimal styling required. A second option is to write your CSS inline. Adding CSS inline means you would literally code style attributes for every individual HTML element. This is generally considered the least favorable approach as it leads to lots of repeated code - not very efficient, and not a cool coder's best practice. :( While the third option - using external CSS files - adds a step to your coding process, it's actually a more efficient practice for larger HTML projects. Using an externall CSS file involves writing a CSS file with all of the element attributes defined, then linking your HTML doc to said CSS file via a link in the head tag of the HTML doc.

In addition to CSS element styles, box sizing and positioning are important to the overall look and feel of a web page. HTML elements are essentially boxes, with each box having 4 components. It's difficult to get a box size just right because there are so many components, so you'll want to use a couple of special techniques to address this.

Box Sizing

One option is to set box sizes in percentages instead of pixels. This is a great set up for responsive design, as the web page will automatically adjust the sizes of your boxes as you scale your page down or up.

Another option is to set the box-sizing attribute to border-box for every element. This will provide a consistent sizing that includes the main aspet of the box you are sizing, rather than having to size each component separately.

Box Positioning

While Divs are block elements (as opposed to inline), they don't have parameters to set their size across a page. By default, they display across an entire page's width. You can add the rule display: flex to override this default behavior and allow Divs to display side-by-side on a page.

Last but not least...

To make it even more interesting, surprise! Different browsers react slightly differently to CSS and HTML, so browsers may display the same code differently. You will want to add descriptors for each browser (webkit, moz, ms) in CSS so each browser will have the info it needs to read your code and display accurately.
